Python Memory Model — Reference vs Value

في Python، المتغيرات هي references وليست containers. يعني لما بتعمل a = b، الاتنين بيشيروا لنفس الـ object في الذاكرة.

a = [1, 2, 3]
b = a          # b يشير لنفس الـ list
b.append(4)
print(a)       # [1, 2, 3, 4] — a اتغيرت!

# للـ immutable types (int, str, tuple): مختلف
x = 5
y = x
y = 10
print(x)       # 5 — x ما اتغيرتش

Type Conversion — تحويل الأنواع

▼

🧠 ما هو؟

تحويل قيمة من Type لنوع آخر. البيانات تأتي غالبًا كنصوص — خاصة من input() أو CSV أو APIs.

❌ بدون Conversion — Error
age = input("Age: ")   # "25" — string!
print(age + 5)
# TypeError: can only concatenate
# str (not "int") to str
✅ مع Conversion — يشتغل
age = int(input("Age: "))  # 25 — int!
print(age + 5)
# Output: 30

Explicit Conversion — التحويل الصريح

دوال التحويل الأساسية
# String → Number
int("5")        # 5
float("3.14")   # 3.14

# Number → String
str(10)         # "10"
str(3.14)      # "3.14"

# → Boolean
bool(1)         # True
bool(0)         # False
bool("hello")   # True
bool("")        # False

# → Integer من Boolean
int(True)      # 1
int(False)     # 0

Implicit Conversion — التحويل التلقائي

Python يحول تلقائيًا في بعض الحالات
result = 5 + 2.5
# Python يحول 5 إلى 5.0 تلقائيًا
print(result)       # 7.5 (float)
print(type(result)) # <class 'float'>

# int + bool = int
print(10 + True)  # 11
print(10 + False) # 10

⚠️ Edge Cases — الحالات الخطيرة

Invalid Conversion
int("hello")
# ValueError: invalid literal
# for int() with base 10: 'hello'

int("3.14")
# ValueError: ده float مش int مباشرة
# الصح: int(float("3.14"))
Precision Loss — فقدان الدقة
int(3.9)   # 3 — مش 4!
int(3.1)   # 3
int(-3.9)  # -3 — مش -4!

# ده truncation وليس rounding
# لو عايز rounding:
round(3.9)  # 4
round(3.5)  # 4 — banker's rounding

Safe Conversion في Data Engineering

# الطريقة الاحترافية — دايمًا use try/except
def safe_int(value, default=None):
    try:
        return int(value)
    except (ValueError, TypeError):
        return default

# استخدام في Pipeline
rows = ["25", "abc", "30", None]
ages = [safe_int(row, default=0) for row in rows]
print(ages)  # [25, 0, 30, 0]

Production Pipeline — Complete Example

import logging

logging.basicConfig(level=logging.INFO)

def parse_record(row: list) -> dict:
    """Parse and validate a CSV row safely."""
    try:
        return {
            "name": str(row[0]).strip(),
            "age": int(row[1]),
            "salary": float(row[2]),
        }
    except (ValueError, IndexError) as e:
        logging.warning(f"Bad record {row}: {e}")
        return None

def validate_record(record: dict) -> bool:
    """Business rules validation."""
    if record is None:
        return False
    if not (0 < record["age"] <= 150):
        return False
    if record["salary"] < 0:
        return False
    return True

# Pipeline
raw_data = [
    ["Ahmed", "25", "5000"],
    ["Ali", "abc", "3000"],   # bad age
    ["Sara", "30", "7000"],
]

valid_records = [
    record for row in raw_data
    if (record := parse_record(row)) and validate_record(record)
]

print(valid_records)
# [{'name': 'Ahmed', 'age': 25, 'salary': 5000.0},
#  {'name': 'Sara', 'age': 30, 'salary': 7000.0}]

💼 Interview Preparation — تحضير المقابلات

▼
🟢 Basic Questions
Q1: ما الفرق بين = و == ؟ ▼

= هو Assignment — بيخزن قيمة في متغير: x = 5

== هو Comparison — بيقارن قيمتين وبيرجع bool: x == 5 → True

🔄 Follow-up: "لو عملت if x = 5: ايه اللي بيحصل؟" → SyntaxError في Python.
Q2: ليه input() بترجع String دايمًا؟ ▼

لأن المستخدم بيدخل نص خام من الـ terminal، واللغة لا تفترض النوع تلقائيًا — المسؤولية على المبرمج.

user_input = input("Enter age: ")
print(type(user_input))  # <class 'str'> دايمًا
Q3: ما الفرق بين / و // في Python؟ ▼

/ True Division — دايمًا بيرجع float: 5 / 2 = 2.5

// Floor Division — بيرجع int ويقرب للأسفل: 5 // 2 = 2

🔄 Follow-up: "ايه نتيجة -5 // 2 ؟" → -3 مش -2 (floor = أصغر integer)
🟡 Tricky Questions
Q4: ما نتيجة bool("False") ؟ ▼

True 🎯

لأن أي String غير فارغ = True — بغض النظر عن محتواه. "False" هي string فيها حروف، مش القيمة المنطقية False.

bool("False")  # True
bool("")      # False — فارغ فقط
bool("0")     # True — غير فارغ!
Q5: ما نتيجة int(True) و int(False) ؟ ▼

int(True) = 1 و int(False) = 0

في Python، bool هو subclass من int. True = 1، False = 0.

print(True + True)    # 2
print(True * 10)     # 10
print(sum([True, False, True, True]))  # 3 — count of True!
Q6: ليه int(3.99) = 3 وليس 4؟ ▼

لأن int() بيعمل truncation (بتر الكسر) وليس rounding (تقريب).

int(3.99)    # 3  — truncation
int(3.1)     # 3  — truncation
int(-3.9)   # -3 — truncation (ليس -4)
round(3.9)  # 4  — rounding
round(3.5)  # 4  — banker's rounding (round to even)
🔴 Advanced / System Thinking
Q7: لو CSV فيه age كنصوص، هتعمل Validation إزاي في Production Pipeline؟ ▼

Senior Answer:

  1. Parse safely: استخدم try/except حول كل conversion
  2. Handle conversion errors: مش بتـbreak — بتـlog وبتـquarantine
  3. Validate range/business rules: 0 < age <= 150
  4. Reject/Quarantine bad records: اعمل Dead Letter Queue
# Production Pattern
valid_records = []
bad_records = []

for row in csv_rows:
    try:
        age = int(row["age"])
        assert 0 < age <= 150, f"Invalid age: {age}"
        valid_records.append(row)
    except (ValueError, AssertionError) as e:
        bad_records.append({"row": row, "error": str(e)})

# Process valid, alert on bad
process(valid_records)
alert_team(bad_records)
Q8: ما هو Dynamic Typing وما مخاطره في Data Engineering؟ ▼

Dynamic Typing = Python يسمح لنفس المتغير يحمل أنواع مختلفة في أوقات مختلفة.

المخاطر في Data Engineering:

  • Schema drift — العمود بييجي int في batch أول وstring في batch تاني
  • Silent bugs — الـ Pipeline مش بتكسر، بس بتعمل نتائج غلط
  • Late failures — الـ Bug بيظهر في آخر الـ Pipeline مش في الأول

الحل: استخدم Type Hints + Pydantic + schema validation في مدخل الـ Pipeline.

Q9: [AI Enhancement] ما الفرق بين is و == في Python؟ ▼

== بيقارن القيم. is بيقارن الهوية في الذاكرة (نفس الـ object).

a = [1, 2, 3]
b = [1, 2, 3]

a == b   # True  — نفس القيمة
a is b   # False — object مختلف في الذاكرة

# الاستخدام الصح لـ is:
x is None   # الصح للـ None check
x == None  # بيشتغل بس أقل Pythonic

🚫 Wrong Answers Alert — أخطاء شائعة في الإنترفيو

❌ غلط "Python يعرف النوع من اسم المتغير" لا — Python يحدد النوع من القيمة المُعيَّنة، مش من الاسم.
❌ غلط "المتغير نفسه بيخزن البيانات" لا — المتغير reference (مؤشر) على القيمة في الذاكرة.
❌ غلط "int(3.9) = 4" لا — int() يعمل truncation مش rounding. النتيجة 3.
